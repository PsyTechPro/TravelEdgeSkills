# Contributing to TravelEdgeSkills

Thank you for wanting to make this better. The goal is to keep every skill specific, actionable, and genuinely useful — not generic travel advice that exists everywhere else.

## What Makes a Good Contribution

### ✅ Good contributions
- Specific program valuations with current data
- Route-specific sweet spots and redemption strategies
- Scripts and exact language that works at hotels, airlines, or card issuers
- New skill files for underserved travel hacking topics
- Corrections when data becomes outdated (award rates change, programs devalue)

### ❌ Not a good fit
- Generic travel tips ("book early!", "be flexible!")
- Affiliate-motivated recommendations
- Anything requiring readers to be wealthy to benefit
- Vague strategies without specific actionable steps

---

## Adding or Improving a Skill File

### Skill File Format

Every skill file should include:

```markdown
# Skill Name

## Purpose
One sentence: what problem does this solve and for whom?

## Trigger Phrases
List of natural language questions this skill should respond to.

## What You Need From the User
What inputs does the skill need to give a good answer?

## Core Workflow / Knowledge
The actual expert knowledge, frameworks, and strategies.

## Output Format (if applicable)
What a good response looks like.
```

### Standards

- **Be specific.** "Transfer to Aeroplan for 45,000 miles one-way business class to Europe" is good. "Use airline points for flights" is not.
- **Include current data.** Point valuations, transfer ratios, and program rules change. Note when data was last verified if possible.
- **No affiliate agenda.** Recommend what's actually best, not what pays a commission.
- **Include the "avoid" list.** Knowing what NOT to do is as valuable as knowing what to do.

## Suggesting a New Skill

Open an issue with:
- **Skill name**
- **The problem it solves** (one sentence)
- **Who benefits** (broad audience, not niche)
- **Why existing skills don't cover it**

Good candidates for new skills:
- Travel insurance optimization
- Status match and challenge strategies
- Rental car hacking (insurance, upgrades, elite status)
- Cruises and points
- International SIM card and data strategies
- Travel during airline disruptions (compensation rights, rebooking)

---

## Reporting Outdated Information

Travel programs change constantly. If you find outdated data:

1. Open an issue with the specific outdated information
2. Include the correct current data with a source if possible
3. Or submit a PR with the correction directly

---

## Pull Request Process

1. Fork the repository
2. Make your changes in a branch
3. Submit a PR with a clear description of what changed and why
4. PRs that improve specificity, accuracy, or add genuinely new strategies are prioritized

---

## Code of Conduct

Keep it helpful and honest. This project exists to democratize knowledge that's been kept obscure for too long. Stay true to that.
